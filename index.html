Web 4.0

What is it?

Web 4.0 is giving data back to the people, without any requirements of crypto. With Web 4.0, you have a singular profile that can exist across the entire web - never making another profile ever again, just connecting your existing one to another website.



How does this give me my data back

You get to not only choose where your profile is stored and backed up to, but also can completely erradicate your internet existance in practically 1 button. With supporting websites, they will store any data they or you create, on said server. Any that choose to host the data themselves, will be upheald to an open source data protection standard, and failing to upkeep it will result in public shaming via black-lists. Search engines will state the website does not properly protect your data to specific standards & will give warnings ahead of time if there are worse issues. Search engine should give the option to automatically block all "unsafe" from popping up.

Ok, but HOW?

By compartmentalizing common sections of the internet into open server software, anyone can spin up a server to host their own data, if they do not want to use another business' services. At the current moment of conception, there are just a few compartmentalizations that can be had.

Dangers

  • Scammers using fake names, potentially stopped by PUIN
  • Scammers trying to get you to backup you profile to their server, potentially only stoppable education (Just like email scams)

Profiles

Profiles are an internet identification server, either hosting your main profile or backup profile. Backup profiles are clones of the main profile, in case the main profile server goes down. Profiles need to at the least host a publically accessible username, a basic list of just servers/websites connected to user, and a PUIN (Public Username Idendification Number) associated with that username. Hidden from the public, the server needs to hold more identifiable information of how it connects to said servers/websites (Such as a private key or such), a private PUIN key unique to your PUID, the ability of multiple authentication methods (auth app, email, and any extras they can afford), and a list of communication specific connections (used for web4.0 communications).

PUIN (Public Username Identification Number) allows a user to publically reserve their name internet wide. One is not required to get one, and can take the username used by another - but it will be marked as not being PUIN registered & potentially not who they say they are (Potentially anti-scammer technique).
For the first 5 years of Web4.0, it should be discouraged for people to sell names off - to reduce mass username creation spamming, so that everyone at the time has a chance to have the name they want.

Digital Ownership Servers

DOServers act as a method of containing a database of digital product ownership. These would contain things like digital games, music, software, and even videos/images that have been posted to websites. Servers are free to send assets to DOs'and can decide to require multi-factor authentication to recieve; but sending digital items must require multi-factor auth. DOs items do not need to store any more than just text, and websites should be encouraged to use DOs made by other services. For exmaple, purchasing a game from Microsoft will deposit a license ID & necessary info with it, and Steam can read from your DOs that game license, check that it is legitmate, and then make a stored reference in Steam servers so that you now have access to the game there. If an item is transfered from a DOs, a message is sent across all the connected servers/websites that it is no longer owner by said user.

Communication Servers

Communication servers are simply, methods of communicating between people; limited to at the least, plain text. These should be considered a direct replacement to text messaging and cell phones (No limits on text/calls, including regional distances).
